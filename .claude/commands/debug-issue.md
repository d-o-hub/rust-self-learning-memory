Debug a runtime issue or unexpected behavior.

Systematic debugging of production or test issues.

Usage: /debug-issue [description of issue]

Steps:
1. Gather information about the issue
2. Create minimal reproduction
3. Add instrumentation and logging
4. Run with RUST_LOG=debug
5. Analyze logs and behavior
6. Form hypothesis about root cause
7. Test hypothesis
8. Apply fix
9. Verify fix resolves issue
10. Add regression test
11. Document findings
12. Report resolution

Use the debugger agent for this workflow.
