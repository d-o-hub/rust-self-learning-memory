1. **Check existing docs first**: Don't duplicate information
2. **Choose appropriate folder**: STATUS/, ROADMAPS/, CONFIGURATION/, etc.

5. **Add metadata**: Include Last Updated, Purpose, Status in header

### Updating Existing Documents

1. **Update timestamps**: Always update "Last Updated" field
2. **Maintain cross-references**: Verify relative paths are correct
3. **Check consolidation**: Can content be merged with related docs?
4. **Review regularly**: Keep information current and accurate

### Archiving Completed Work

1. **Choose archive location**:
   - archive/completed/YYYY-MM/ for finished work
   - archive/goap-plans/YYYY-MM/ for GOAP plans
   - archive/releases/vX.X.X/ for version-specific docs
2. **Update references**: Remove from active navigation
3. **Preserve context**: Maintain file relationships in archive
4. **Update archive index**: Document in appropriate index file

### Consolidating Documents

1. **Identify redundancy**: Find overlapping or similar content
2. **Create comprehensive guide**: Merge into single authoritative document
3. **Preserve all information**: Don't lose valuable content
4. **Update references**: Point to new consolidated location
5. **Archive originals**: Move source files to archive/
