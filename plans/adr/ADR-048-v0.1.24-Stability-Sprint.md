# ADR-048: v0.1.24 Stability & Hygiene Sprint

- **Status**: Proposed
- **Date**: 2026-03-30
- **Deciders**: Project maintainers
- **Related**: ADR-047 (v0.1.22 Quality), ADR-029 (GH Actions Modernization), ADR-041 (Test Health)
- **Workspace Version**: 0.1.23 (to be bumped to 0.1.24 at sprint close)

## Context

With v0.1.23 remediation complete (WG-051–WG-058), the project has strong feature coverage and durable storage. However, a 2026-03-30 audit reveals stability and hygiene gaps that should be addressed before the next feature sprint:

1. **Test instability**: Two test failures in the full suite — `benchmark_memory_usage` (DBSCAN time budget exceeded at 103s) and `quality_gate_performance_regression` (120s timeout running cargo test inside a test)
2. **Dependency drift**: 2 open Dependabot PRs (9 Rust patch-minor + 3 GitHub Actions major bumps)
3. **Plans/docs drift**: STATUS/CURRENT.md still references v0.1.22 as released version; ACTIONS.md has 9 actions marked "Pending" that were completed; ignored test count documented as 124 but actual is 114
4. **Stale code annotations**: WG-018/WG-019/WG-020/WG-021 from v0.1.19 sprint remain formally "Pending" in GOALS.md despite partial resolution
5. **GitHub Actions modernization**: actions/checkout v4→v6 (Node.js 24), codecov/codecov-action v5→v6 — major version bumps that need validation

## Decision

Structure v0.1.24 as a stability-and-hygiene sprint with 3 phases:

### Phase 1: Test Stability (P0)

| ID | Goal | Details |
|----|------|---------|
| WG-059 | Fix benchmark_memory_usage DBSCAN time budget | Increase CI time budget or reduce dataset size; the test asserts `duration < 60s` in CI but takes 103s |
| WG-060 | Fix quality_gate_performance_regression timeout | This quality gate test runs `cargo test` as a subprocess — inherently slow. Mark as `#[ignore]` with reason or refactor to avoid subprocess invocation |

### Phase 2: Dependency & CI Modernization (P1)

| ID | Goal | Details |
|----|------|---------|
| WG-061 | Merge Dependabot Rust patch-minor PR #403 | 9 crate updates — review and merge after CI passes |
| WG-062 | Merge Dependabot GitHub Actions PR #402 | actions/checkout v4→v6, codecov v5→v6, wait-on-check v1.5→v1.6 — major version bumps requiring workflow validation |
| WG-063 | Validate CI after actions upgrades | Ensure all 13 workflows pass after checkout v6 + codecov v6 |

### Phase 3: Plans & Documentation Hygiene (P2)

| ID | Goal | Details |
|----|------|---------|
| WG-064 | Sync STATUS/CURRENT.md to v0.1.23 | Update released version, workspace version, test counts, ignored test count (114 not 124) |
| WG-065 | Close stale ACTIONS.md pending items | ACT-038–046 already complete — mark them as such; ACT-017 should be resolved |
| WG-066 | Resolve stale GOALS.md WGs | WG-008 (revised target met), WG-018 (31 ≤40 target met), WG-019 (assess), WG-020 (done), WG-021 (assess) — update statuses |
| WG-067 | Update GOAP_STATE.md and ROADMAP_ACTIVE.md | Reflect v0.1.24 sprint goals, link to this ADR and execution plan |

## Quality Gates

All must pass before v0.1.24 tag:

- `./scripts/code-quality.sh fmt`
- `./scripts/code-quality.sh clippy --workspace`
- `cargo nextest run --all` (0 failures, 0 timeouts)
- `cargo test --doc` (0 failures)
- `./scripts/quality-gates.sh`

## Consequences

### Positive
- CI becomes fully green with 0 failures/timeouts
- GitHub Actions use current major versions (checkout v6, codecov v6)
- Plans/ folder accurately reflects implementation reality
- Stale WGs formally closed — reduces confusion in future sprint planning

### Negative
- No new features in this sprint (pure stability)
- Benchmark time budget increase may mask future performance regressions

### Risks
- **actions/checkout v6**: Requires Actions Runner v2.329.0+ — verify GitHub-hosted runners are compatible
- **codecov v6**: Requires Node.js 24 support — verify runner compatibility
