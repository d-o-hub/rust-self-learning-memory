## Archive Maintenance

### When to Archive
- Implementation complete → Move to `completed/`
- Version released → Move to `releases/vX.Y.Z/`
- Superseded by new doc → Move to `legacy/`
- Research completed → Move to `research/`

### Archive Lifecycle
1. **Active**: Files in plans/ root (current work)
2. **Archived**: Moved to appropriate archive category
3. **Indexed**: Added to this index with description
4. **Preserved**: Historical context maintained indefinitely

### Access Guidelines
- **Recent work**: Check active plans/ files
- **Historical context**: Search archive by date/category
- **Version history**: Check releases/vX.Y.Z/
- **Research findings**: Check research/ folder
